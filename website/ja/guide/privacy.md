# プライバシー

VMarkはあなたのプライバシーを尊重します。何が起きているか — そして何が起きていないか — を正確にお伝えします。

## VMarkが送信するもの

VMarkには**自動更新チェッカー**が含まれており、定期的にサーバーに接続して新しいバージョンが利用可能かどうかを確認します。これがVMarkが行う**唯一**のネットワークリクエストです。

各チェックで送信されるのは正確に以下のフィールドだけです — それ以上はありません:

| データ | 例 | 目的 |
|------|---------|---------|
| IPアドレス | `203.0.113.42` | HTTPリクエストに固有のもの — 受け取らないようにすることはできません |
| OS | `darwin`、`windows`、`linux` | 正しい更新パッケージを提供するため |
| アーキテクチャ | `aarch64`、`x86_64` | 正しい更新パッケージを提供するため |
| アプリバージョン | `0.5.10` | 更新が利用可能かどうかを判断するため |
| マシンハッシュ | `a3f8c2...`（64文字の16進数） | 匿名デバイスカウンター — ホスト名+OS+アーキテクチャのSHA-256。逆算不可能 |

完全なURLは次のようになります:

```
GET https://log.vmark.app/update/latest.json?target=darwin&arch=aarch64&version=0.5.10
X-Machine-Id: a3f8c2b1d4e5f6078a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3d4e5f6a7b8c9d0e1
```

自分で確認できます — エンドポイントは[`tauri.conf.json`](https://github.com/xiaolai/vmark/blob/main/src-tauri/tauri.conf.json)にあります（`"endpoints"`を検索）、ハッシュは[`lib.rs`](https://github.com/xiaolai/vmark/blob/main/src-tauri/src/lib.rs)にあります（`machine_id_hash`を検索）。

## VMarkが送信しないもの

- あなたのドキュメントまたはその内容
- ファイル名またはパス
- 使用パターンや機能分析
- いかなる種類の個人情報も
- クラッシュレポート
- キーストロークまたは編集データ
- 逆算可能なハードウェア識別子またはフィンガープリント
- マシンハッシュは一方向のSHA-256ダイジェストです — ホスト名やその他の入力を復元するために逆算することはできません

## データの使用方法

更新チェックログを集計して、[ホームページ](/)に表示されるライブ統計を生成します:

| メトリクス | 計算方法 |
|--------|-------------------|
| **ユニークデバイス** | 1日/週/月ごとの異なるマシンハッシュの数 |
| **ユニークIP** | 1日/週/月ごとの異なるIPアドレスの数 |
| **ピング** | 更新チェックリクエストの総数 |
| **プラットフォーム** | OS+アーキテクチャの組み合わせごとのピング数 |
| **バージョン** | アプリバージョンごとのピング数 |

これらの数字は[`log.vmark.app/api/stats`](https://log.vmark.app/api/stats)で公開されています。隠しているものは何もありません。

**重要な注意事項:**
- ユニークIPは実際のユーザー数を過少カウントします — 同じルーター/VPNの背後にいる複数の人は1人としてカウントされます
- ユニークデバイスはより正確なカウントを提供しますが、ホスト名の変更や新しいOSのインストールでは新しいハッシュが生成されます
- ピングは実際のユーザー数を過大カウントします — 1人が1日に複数回チェックする場合があります

## データ保持

- ログは標準的なアクセスログ形式でサーバーに保存されます
- ログファイルは1MBでローテートされ、最新の3ファイルのみ保持されます
- ログは誰とも共有されません
- アカウントシステムはありません — VMarkはあなたが誰かを知りません
- マシンハッシュはどのアカウント、メール、またはIPアドレスにも関連付けられていません — 匿名デバイスカウンターのみです
- トラッキングクッキー、フィンガープリンティング、または分析SDKは使用していません

## オープンソースによる透明性

VMarkは完全なオープンソースです。ここに記載されているすべてのことを確認できます:

- 更新エンドポイントの設定: [`src-tauri/tauri.conf.json`](https://github.com/xiaolai/vmark/blob/main/src-tauri/tauri.conf.json)
- マシンハッシュの生成: [`src-tauri/src/lib.rs`](https://github.com/xiaolai/vmark/blob/main/src-tauri/src/lib.rs) — `machine_id_hash`を検索
- サーバー側の統計集計: [`scripts/vmark-stats-json`](https://github.com/xiaolai/vmark/blob/main/scripts/vmark-stats-json) — [公開統計](https://log.vmark.app/api/stats)を生成するためにサーバーで実行される正確なスクリプト
- コードベースに他のネットワーク呼び出しは存在しません — `fetch`、`http`、または`reqwest`を自分で検索してください

## 更新チェックの無効化

自動更新チェックを完全に無効にしたい場合は、ネットワークレベル（ファイアウォール、`/etc/hosts`、またはDNS）で`log.vmark.app`をブロックできます。VMarkはそれなしでも正常に動作し続けます — 更新通知を受け取らないだけです。
