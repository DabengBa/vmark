# App Internationalization Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full i18n support to the VMark desktop app — React frontend (react-i18next) and Rust backend (rust-i18n) — for 10 languages.

**Architecture:** react-i18next with namespace-split JSON files for the frontend, rust-i18n crate with YAML files for Rust menus. Language stored in settings, menu rebuilt at runtime on locale change. macOS menu subsystem migrated from title-based to ID-based lookup as a prerequisite.

**Tech Stack:** react-i18next, i18next, i18next-resources-to-backend, @i18next/cli, rust-i18n v3, Claude API for AI translation.

**Spec:** `docs/superpowers/specs/2026-03-16-internationalization-design.md`

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import resourcesToBackend from "i18next-resources-to-backend";
import { useSettingsStore } from "@/stores/settingsStore";

const localeModules = import.meta.glob("./locales/*/*.json");

i18n
  .use(initReactI18next)
  .use(
    resourcesToBackend((lng: string, ns: string) => {
      const key = `./locales/${lng}/${ns}.json`;
      const loader = localeModules[key];
      if (!loader) return Promise.reject(new Error(`Missing locale: ${key}`));
      return loader() as Promise<{ default: Record<string, string> }>;
    })
  )
  .init({
    lng: useSettingsStore.getState().general.language,
    fallbackLng: {
      "zh-TW": ["zh-CN", "en"],
      "pt-BR": ["en"],
      default: ["en"],
    },
    load: "currentOnly",
    ns: ["common", "menu", "statusbar"],
    defaultNS: "common",
    interpolation: {
      escapeValue: false,
    },
  });

i18n.on("languageChanged", (lng) => {
  document.documentElement.lang = lng;
});

export default i18n;
```
