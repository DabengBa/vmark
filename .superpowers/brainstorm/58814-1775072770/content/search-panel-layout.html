Where should the search panel live?

Consider how it fits into VMark's existing layout (sidebar + editor + outline)

A

Sidebar Panel (VS Code style)

Replaces file explorer in the left sidebar. Search input at top, grouped results below. Persists while you navigate files. Clicking a result opens the file in editor.

🔍 search query...
☐ Case ☐ Word ☐ Regex
notes.md
3: ...matching line...
17: ...another line...
readme.md
1: ...line here...
Editor area
B

Center Overlay (Spotlight style)

Like Quick Open — appears as a floating overlay on Cmd+Shift+F, disappears when you select a result. Lightweight, fast, non-persistent. Results listed below the search input.

🔍 search query...
☐ Case ☐ Word ☐ Regex   3 results in 2 files
notes.md:3 — ...matching line...
notes.md:17 — ...another line...
readme.md:1 — ...line here...
C

Bottom Panel

Horizontal panel below the editor (like terminal in VS Code). Search input on the left, results on the right. Can be resized or collapsed. Persists across file switches.
